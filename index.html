TypeScript For Dummies

TypeScript in its most basic form is JavaScript with the ability to make variables strongly typed. This allows for the creation of JavaScript like applications that have more strict typing similar to what is found in C#, Java, etc.

TypeScript Basics

TypeScript is able to use type inference to ensure that code later on is following the same type for the assigned variable. An example of creating a basic string variable is shown below using the : operator

let exampleStringVariable:string = "This is a String";
//or
let exampleStringVariable:string = 'This is a String';

TypeScript also allows for inference of types at the time of initialization by usign the let keyword. The code below is infered to have a type of variableName:string due to being set to a string value at initialization.

let variableName = "This is an inferred variable";

Creating Types

TypeScript allows for the creation of "types" which are essentially object definitions in other programming lnaguages. This allows for "objects" that are more strictly defined than typical JSON objects. The definition of types is the same as the definition of JSON objects, an examples is shown below.

const testObject {
 objName: "String Object Name",
 objId: 100
}

TypeScript also allows for the creation of interfaces through the interface keyword. Interfaces help provide warnings, but will not cause a crash if not followed.

interface Person{
 name: string,
 age: number
}

Creating Classes

TypeScript also allows for the creation of classes with defined tyes for constructors and methods as shown below in the code example from TypeScript for JavaScript Programmers

interface User {
 name: string;
 id: number;
}

class UserAccount {
 name: string;
 id: number;
 constructor(name: string, id: number) {
  this.name = name;
  this.id = id;
 }
}

const user: User = new UserAccount("Murphy", 1);

Functions can also have types defined for both return and parameters for functions

function functionWithReturn(): string { }

function functionWithParam(name: string) { }

Union Types

TypeScript allows programers to define their own "types" through the sue of the type keyword and | union operator. These programmer defined types are in a lot of ways similar to defining enumaerations in C# and Java. An example of a few programmer designed types are shown below.

type booleanType = true | false;
type allowedNames = "Jane" | "John" | "Daniel" | "Beth";
type multiplesOfFourToTwenty = 0 | 4 | 8 | 12 | 16 | 20;

The union operator (|) can also be used to allow a funciton to accept multiple types as input. For example the code below would allow for both a string or a number to be taken as the parameter

function stringNum(val: string | number) { }

the typeof keyword is used often in TypeScript to help determine if an input variable is of the correct type. Ane example of checking if the type is a string is shown below

if(typeof obj === "string") { //do something }

Generics

TypeScript allowsfor creation of generic types, just like C# and Java. These generics can be used similarly to Java to help create interfaces and objects. An example of a generic interface is shown below.

interface GenericInterface <Type>
 add: (obj: Type) => void;
}

declare const genericDec: GenericInterface<string>

const obj = genericDec.get(); // obj is a string due to above decleration

Structural Typing

At the core of TypeScript is its structural typing system. Structural typing is based on the "shape" of a value. The shape can generally be thought of as two objects having the same fields being the same type of object. It is better shown through the example below. Note that an interface or a class could be used interchangably in this example

interface User { // Interface for a User Object
 name: string;
 id: number;
}

// Creates a user object through structural typing
const exampleUser = { name: "Daniel", id: 1234 }
/* even though exampleUser isn't explicitly told to be a User object it will become one based on the shape */
